# deploy-and-test — Деплой конфигурации и запуск тестов

Автоматизация цикла: загрузка конфигурации из XML → обновление БД → запуск тестов.

## Предварительные условия

- Файл `infobasesettings.md` с параметрами базы (создать если нет)
- Платформа 1С:Предприятие 8.3 с доступным DESIGNER

### Формат infobasesettings.md

```markdown
- Строка подключения: /S server/basename (серверная) или /F C:\Path\To\Base (файловая)
- URL публикации: http://localhost/base/
- Путь к платформе: C:\Program Files\1cv8\8.3.XX.YYYY\bin\1cv8.exe
```

---

## Шаги

### 1. Загрузка конфигурации из XML

```powershell
& "<путь к 1cv8.exe>" DESIGNER <строка подключения> /DisableStartupMessages /LoadConfigFromFiles <корень проекта> /Out <путь к логу>
```

Прочитать лог — убедиться в успешной загрузке.
Подождать 5-10 секунд.

### 2. Обновление структуры БД

```powershell
& "<путь к 1cv8.exe>" DESIGNER <строка подключения> /DisableStartupMessages /UpdateDBCfg -Dynamic+ -SessionTerminate force /Out <путь к логу>
```

Прочитать лог — убедиться в успешном обновлении.

### 3. Запуск тестов

Если URL публикации указан — открыть для тестирования через веб-клиент.
Если установлен YaXUnit или vanessa-runner — запустить тесты:

```powershell
& "<путь к 1cv8.exe>" ENTERPRISE <строка подключения> /DisableStartupMessages /Execute <путь к тестам> /Out <путь к логу>
```

### 4. Отчёт

Вывести результат:
- Загрузка конфигурации: OK / ОШИБКА
- Обновление БД: OK / ОШИБКА
- Тесты: ПРОЙДЕНЫ / ОШИБКИ / НЕ ЗАПУЩЕНЫ

---

## Параметры из infobasesettings.md

При первом вызове — создать `infobasesettings.md` в корне проекта, запросив у пользователя:
1. Строку подключения к базе
2. URL публикации (опционально)
3. Путь к платформе (если отличается от стандартного)

---

## Замечания

- Путь к корню проекта — текущая рабочая директория
- Лог записывается во временный файл, читается после каждого шага
- При ошибке на любом шаге — остановить выполнение и показать лог
- Флаг `-Dynamic+` позволяет динамическое обновление; убрать если нужна полная реструктуризация
