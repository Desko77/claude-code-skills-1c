# Типовые механизмы ЗУП 3.1 - кадровый учет и расчет

Универсальный справочник API конфигурации 1С:ЗУП 3.1 для работы с кадровыми данными.
Применяется: при разработке на конфигурациях ЗУП 3.1 (базовая, ПРОФ, КОРП).

---

## 1. Архитектура: Сотрудник vs ФизическоеЛицо

| | Справочник.Сотрудники | Справочник.ФизическиеЛица |
|---|---|---|
| **Хранит** | Рабочее место (трудовой договор) | Персональные данные человека |
| **Данные** | Организация, должность, подразделение, оклад, график | ФИО, дата рождения, паспорт, ИНН, СНИЛС |
| **Связь** | Реквизит `ФизическоеЛицо` -> ФизическиеЛица | Один ФЛ -> несколько Сотрудников |

**Правило:** данные о рабочем месте -> Сотрудник; персональные данные -> ФизическоеЛицо.

---

## 2. Иерархия модулей

Публичный API - модули **без** суффикса "Внутренний". Методы из `ПрограммныйИнтерфейс` сохраняют совместимость.

```
КадровыйУчет (~275 экспортных методов) - основной API
  -> КадровыйУчетВнутренний (НЕ вызывать напрямую, но содержит полные списки полей)
  -> КадровыйУчетРасширенный (~244 метода) - КОРП: стажи, подработки, ОсновнойСотрудник
  -> КадровыйУчетПовтИсп (кешируемые)
КадровыйУчетВызовСервера - серверные вызовы с клиента
КадровыйУчетКлиентСервер - клиент-серверные
ЗарплатаКадры (~310 методов) - организации, валюта, ответственные лица
ЗарплатаКадрыОбщиеНаборыДанных - отборы, ВТ, механизм представлений СКД
ФизическиеЛицаЗарплатаКадры / ...КлиентСервер - ФИО, склонение, пол
```

## 3. КадровыеДанныеСотрудников - главный метод

```bsl
// Массовый (основной)
ТЗ = КадровыйУчет.КадровыеДанныеСотрудников(ТолькоРазрешенные, СписокСотрудников,
    КадровыеДанные, ДатаПолученияДанных, ПоляОтбора, ФормироватьСПериодичностьДень);
// Возвращает: ТаблицаЗначений (Сотрудник, ФизическоеЛицо, Период + запрошенные поля)

// Для одного (обертка, возвращает СтрокаТЗ или Неопределено)
Данные = КадровыйУчет.КадровыеДанныеСотрудника(ТолькоРазрешенные, Сотрудник, КадровыеДанные, Дата);
```

**Параметры:**
- `ТолькоРазрешенные` (Булево) - учитывать RLS. `Ложь` = привилегированный доступ
- `СписокСотрудников` - Массив СправочникСсылка.Сотрудники или одна ссылка
- `КадровыеДанные` - Строка через запятую: `"Должность, Подразделение, ДатаПриема"`
- `ДатаПолученияДанных` - Дата; `'00010101'` = последние данные без ограничения по дате

## 5. Справочник полей КадровыеДанные

### 5.1. Поля сотрудников - периодические (зависят от даты)
`Организация`, `Подразделение`, `Должность`, `ДолжностьПоШтатномуРасписанию`, `ВидЗанятости`, `ВидДоговора`, `КоличествоСтавок`, `МестоВСтруктуреПредприятия`, `ВидСобытия`, `Состояние`, `СостояниеПериодРегистрации`, `Территория`, `ТерриторияВыполненияРаботВОрганизации`, `РазрядКатегория`, `РайонныйКоэффициент`, `РайонныйКоэффициентРФ`, `ГрафикРаботы`, `СуммированныйУчетРабочегоВремени`, `СреднемесячноеЧислоЧасов`, `ДоляНеполногоРабочегоВремени`, `ТарифнаяСтавка`, `ФОТ`, `Надбавка`, `ОсновноеНачисление`, `СовокупнаяТарифнаяСтавка`, `СпособРасчетаАванса`, `Аванс`, `ЗарплатныйПроект`, `НомерЛицевогоСчета`, `СтатьяФинансирования`, `СпособОтраженияЗарплатыВБухучете`, `ДатаДоговораКонтракта`, `НомерДоговораКонтракта`, `ДатаНачала`, `ТрудовойДоговорДатаОкончания`, `СрочныйДоговор`, `КлассУсловийТруда`, `ИспытательныйСрокДатаЗавершения`, `НаходитсяНаИспытательномСроке` (КОРП)

### 5.2. Поля сотрудников - постоянные (из РС ТекущиеКадровыеДанныеСотрудников)
`ДатаПриема`, `ДатаУвольнения`, `ДатаЗавершенияРаботы`, `ТекущаяОрганизация`, `ТекущееПодразделение`, `ТекущаяДолжность`, `ТекущийВидЗанятости`, `ОсновноеРабочееМестоВОрганизации`, `ОформленПоТрудовомуДоговору`, `ТекущаяТарифнаяСтавка`, `ТекущийФОТ`, `ТабельныйНомер`, `ПриказОПриеме`, `ПриказОПриемеДата`, `ПриказОПриемеНомер`, `ПриказОбУвольнении`, `ПриказОбУвольненииДата`, `ПриказОбУвольненииНомер`, `ПриказОбУвольненииСтатьяТКРФ`, `РуководительСтруктурногоПодразделения`

### 5.3. Поля физических лиц - периодические
**ФИО:** `Фамилия`, `Имя`, `Отчество`, `ФИОПолные`, `ФамилияИО`, `ИОФамилия`, `Инициалы`
**Гражданство:** `Страна`, `ИННВСтранеГражданства`
**Документ УЛ:** `ДокументВид`, `ДокументСерия`, `ДокументНомер`, `ДокументДатаВыдачи`, `ДокументСрокДействия`, `ДокументКемВыдан`, `ДокументКодПодразделения`, `ДокументПредставление`
**Инвалидность:** `Инвалидность`, `ИнвалидностьДатаВыдачи`, `ИнвалидностьСрокДействияСправки`
**Стажи:** для каждого вида (ОбщийСтаж, СтраховойСтаж, СеверныйСтаж, НепрерывныйСтаж и др.) доступны суффиксы `*Лет`, `*Месяцев`, `*Дней`, `*Вид`, `*ПериодРегистрации`, `*РазмерДней`, `*РазмерМесяцев`
**Воинский учет:** `ВоинскийУчетКатегорияЗапаса`, `ВоинскийУчетЗвание`, `ВоинскийУчетВУС`, `ВоинскийУчетГодность`

### 5.4. Поля физических лиц - постоянные
`ДатаРождения`, `Пол`, `Возраст`, `ИНН`, `СтраховойНомерПФР`, `СтатусНалогоплательщика`, `ДатаСмерти`
**Контакты:** `АдресМестаПроживания(Представление)`, `АдресПоПрописке(Представление)`, `ТелефонМобильный(Представление)`, `EMail(EMailПредставление)`
**Образование:** `Образование1*`, `Образование2*` (УчебноеЗаведение, Специальность, ДатаВыдачи...)
**Семья:** `КоличествоДетей`, `КоличествоИждивенцев`

### Нюанс: периодические vs постоянные
- Без префикса "Текущий" = **периодические** (зависят от ДатаПолученияДанных)
- С префиксом "Текущий" = **постоянные** (всегда актуальные)
- Полный список полей: `КадровыйУчетВнутренний.ЗапросВТКадровыеДанныеСотрудников` / `...ФизическихЛиц`

## 6. Дата приема, увольнения, периоды работы

```bsl
// Через КадровыеДанные (основной способ)
ТЗ = КадровыйУчет.КадровыеДанныеСотрудников(Ложь, Массив, "ДатаПриема, ДатаУвольнения, ДатаЗавершенияРаботы");
```

**Важно!** `ДатаУвольнения` = ГОЛОВНОГО сотрудника. `ДатаЗавершенияРаботы` = ТЕКУЩЕГО (для подработок отличается). Пустая = активен.

```bsl
// Периоды работы (ТЗ: Сотрудник, ДатаПриема, ДатаЗавершенияРаботы)
ПериодыРаботы = КадровыйУчет.ПериодыРаботыСотрудников(МассивСотрудников);

// Дата приема ФЛ (минимальная среди всех рабочих мест)
ДатаПриема = КадровыйУчетРасширенный.ДатаПриемаФизическогоЛица(ФизЛицо, ДатаСведений);

// Первый ли прием на работу?
Первый = КадровыйУчет.ЭтоНачалоТрудовойДеятельности(ФизЛицо, ДатаСобытия, ИсключаемыеРегистраторы);
```

## 7. Стажи

```bsl
// Простой способ - через КадровыеДанныеФизическихЛиц
ТЗ = КадровыйУчет.КадровыеДанныеФизическихЛиц(Ложь, МассивФЛ,
    "ОбщийСтажЛет, ОбщийСтажМесяцев, ОбщийСтажДней, СтраховойСтажЛет", ДатаРасчета);

// Детальный способ - все виды стажа (Соответствие[Сотрудник -> Соответствие[ВидСтажа -> Структура]])
ТЗ = Новый ТаблицаЗначений; // колонки: Сотрудник, Период
Стажи = КадровыйУчетРасширенный.СведенияОСтажахСотрудников(ТЗ);
// Или по ФЛ:
Стажи = КадровыйУчетРасширенный.СведенияОСтажахФизическихЛиц(ТЗ); // колонки: ФизическоеЛицо, Период

// Представление стажа
Текст = КадровыйУчетРасширенный.ПредставлениеСтажа(Дней, Месяцев, Лет); // "26 дней 9 месяцев 40 лет"
Текст = ЗарплатаКадрыРасширенныйКлиентСервер.ПредставлениеСтажа(Лет, Месяцев, Дней); // "40 лет, 9 месяцев и 26 дней"

// Расчет продолжительности стажа
ДанныеСтажа = ЗарплатаКадрыРасширенныйКлиентСервер.СведенияОСтаже();
// заполнить ДатаОтсчета, РазмерМесяцев, РазмерДней...
Продолжительность = ЗарплатаКадрыРасширенныйКлиентСервер.ПродолжительностьСтажа(ДанныеСтажа, ДатаОкончания);

// Нормализация (объединение)
Лет = КадровыйУчетРасширенныйКлиентСервер.ЛетПриПодсчетеСтажа(Лет, Месяцев, Дней);
Месяцев = КадровыйУчетРасширенныйКлиентСервер.МесяцевГодаПриПодсчетеСтажа(Месяцев, Дней);
Дней = КадровыйУчетРасширенныйКлиентСервер.ДнейМесяцаПриПодсчетеСтажа(Дней);
```

## 8. Связь Сотрудник <-> ФизическоеЛицо

```bsl
// Сотрудник -> ФЛ
ФЛ = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Сотрудник, "ФизическоеЛицо");
МассивФЛ = КадровыйУчет.ФизическиеЛицаСотрудников(МассивСотрудников); // уникальные ФЛ

// ФЛ -> все сотрудники (включая уволенных)
ТЗ = КадровыйУчет.СотрудникиФизическихЛиц(МассивФЛ); // ТЗ: Сотрудник, ФизическоеЛицо

// ФЛ -> основной сотрудник (кешируется через КадровыйУчетПовтИсп)
Сотрудник = КадровыйУчет.ОсновнойСотрудникФизическогоЛица(ФизЛицо, Организация, Период);

// ФЛ -> массив основных сотрудников (массовый, приоритет: ОсновноеМесто -> Совместительство)
ТЗ = КадровыйУчет.ОсновныеСотрудникиФизическихЛиц(СписокФЛ, ТолькоРазрешенные, Организация, Период);

// ФЛ -> все сотрудники в организации (КадровыйУчетРасширенный)
Массив = КадровыйУчетРасширенный.МассивСотрудников(ФизЛицо, Организация, Период,
    ВключаяРанееУволенных, РаботникиПоДоговорамГПХ, ПодработкиПоТД, ПоГоловной);
Массив = КадровыйУчетРасширенный.СотрудникиФизическихЛиц(МассивФЛ, Организация, ПоГоловной, ГПХ, Подработки);

// ФЛ -> основной с выводом ошибки
Сотр = КадровыйУчетРасширенный.ОсновнойСотрудникФизЛица(ФизЛицо, Организация, Дата, ВыводитьСообщения);

// Кадровые данные основного сотрудника ФЛ (ищет: Основное -> Совместительство -> уволенный)
Данные = КадровыйУчет.КадровыеДанныеОсновногоСотрудникаФизическогоЛица(
    Организация, ФизЛицо, "Должность, Подразделение", Дата, ТолькоРазрешенные, ТекстОшибки);

// Неоформленные (без трудового договора, при необходимости создает)
Соответствие = КадровыйУчет.НеоформленныеСотрудникиФизическихЛиц(МассивФЛ, Организация);
```

## 9. Получение списков сотрудников организации

```bsl
Параметры = КадровыйУчет.ПараметрыПолученияСотрудниковОрганизацийПоСпискуФизическихЛиц();
Параметры.Организация = Организация;
Параметры.НачалоПериода = НачалоМесяца(Дата);
Параметры.ОкончаниеПериода = КонецМесяца(Дата);
Параметры.КадровыеДанные = "Должность, Подразделение, ДатаПриема";
// Опционально:
Параметры.Подразделение = Подразделение;  // с иерархией
Параметры.РаботникиПоТрудовымДоговорам = Истина;     // Истина/Ложь/Неопределено
Параметры.РаботникиПоДоговорамГПХ = Неопределено;
Параметры.ПодработкиРаботниковПоТрудовымДоговорам = Ложь;
Параметры.ВключаяУволенныхНаНачалоПериода = Ложь;
Параметры.ОтбиратьПоГоловнойОрганизации = Ложь;
// Дополнительные отборы:
ЗарплатаКадрыОбщиеНаборыДанных.ДобавитьВКоллекциюОтбор(
    Параметры.Отборы, "ФизическоеЛицо", "=", ФизЛицо);

Сотрудники = КадровыйУчет.СотрудникиОрганизации(Истина, Параметры);
// Возвращает ТЗ: Период, Сотрудник, ФизическоеЛицо + кадровые данные
```

```bsl
// ФЛ, работавшие в организации за период
ТЗ = КадровыйУчет.ФизическиеЛицаРаботавшиеВОрганизации(Ложь, Организация, Начало, Конец, Подразделение);

// Проверка работающих (для ОбработкаПроверкиЗаполнения)
КадровыйУчет.ПроверитьРаботающихСотрудников(СписокСотрудников, Параметры, Отказ, ПараметрыСообщения);

// Среднесписочная численность
Численность = КадровыйУчет.СреднесписочнаяЧисленностьРаботающих(Организация, Начало, Конец);
```

## 10. Временные таблицы (для пакетных запросов)

```bsl
// Вариант 1: по массиву сотрудников
КадровыйУчет.СоздатьНаДатуВТКадровыеДанныеСотрудников(МВТ, Истина, Массив, "Должность, Подразделение", Дата);
// Создает ВТКадровыеДанныеСотрудников

// Вариант 2: через Описатель (когда сотрудники уже в ВТ)
Описатель = КадровыйУчет.ОписательВременныхТаблицДляСоздатьВТКадровыеДанныеСотрудников(МВТ, "ВТМоиСотрудники");
КадровыйУчет.СоздатьВТКадровыеДанныеСотрудников(Описатель, Истина, "Организация, Подразделение");
```

## 14. ФИО: инициалы, склонение, пол

```bsl
// Фамилия И.О.
ФамилияИО = ФизическиеЛицаЗарплатаКадрыКлиентСервер.ФамилияИнициалы(ФизЛицо);
// И.О. Фамилия
ИОФамилия = ФизическиеЛицаЗарплатаКадры.РасшифровкаПодписи(ДанныеФИО);
// Инициалы
Иниц = ФизическиеЛицаЗарплатаКадрыКлиентСервер.ИнициалыПоИмениОтчеству(Имя, Отчество);

// Склонение ФИО (НомерПадежа: 1-6; Пол: 1-муж, 2-жен)
ФизическиеЛицаЗарплатаКадры.Просклонять(ФИОСтрока, НомерПадежа, РезультатСклонения, Пол);
// или
ПросклоненноеФИО = ЗарплатаКадрыОтчеты.ПросклоненныеФИО(ФИОПолные, НомерПадежа, Пол);
```

## 29. Механизм представлений СКД (для отчетов)

Система виртуальных таблиц: запросы-заглушки `Представления_*` подменяются реальными запросами к регистрам.

```bsl
// Подключение к отчету (в модуле объекта):
Процедура ОпределитьНастройкиФормы(Форма, КлючВарианта, Настройки) Экспорт
    Настройки.События.ПередЗагрузкойНастроекВКомпоновщик = Истина;
КонецПроцедуры

Процедура ПередЗагрузкойНастроекВКомпоновщик(Контекст, КлючВарианта, ...) Экспорт
    ЗарплатаКадрыОбщиеНаборыДанных.ЗаполнитьОбщиеИсточникиДанныхОтчета(СхемаКомпоновкиДанных);
КонецПроцедуры
```

**Типы виртуальных таблиц:** `Представления_КадровыеДанныеСотрудников`, `Представления_КадровыеДанныеФизическихЛиц`, `Представления_СрезПоследних_<ИмяРегистра>`, `Представления_ТаблицаРегистра_<ИмяРегистра>`, `Представления_Периоды`, `Представления_СотрудникиОрганизации`, `Представления_ШтатноеРасписание`, `Представления_ОтработанноеВремя` и 30+ других.

**Нюансы:** итоговая ВТ без подчеркивания (`ПредставленияКадровыеДанныеСотрудников`). Опции в блоке `ГДЕ`: `"ТолькоРазрешенные" = ИСТИНА`. Подробные примеры - `templatesearch "представления СКД ЗУП"`.

Списки доступных полей: `КадровыйУчет.ПоляПредставленийКадровыхДанныхСотрудников()`, `...ФизическихЛиц()`.

## 36. Средний заработок: данные и регистры

## 37. Средний заработок: перерасчет

`РегистрСведений.ПерерасчетСреднегоЗаработка` (независимый, непериодический) помечает документы среднего заработка к пересчету. Измерения: `Организация`, `Сотрудник`, `ДокументСреднегоЗаработка`; реквизит `ДокументОснование`.

```bsl
// Явная регистрация необходимости перерасчета
// ДанныеДляРегистрацииПерерасчетов - ВТ (Организация, Сотрудник, ДокументСреднегоЗаработка, ДокументОснование)
УчетСреднегоЗаработкаСобытия.ЗарегистрироватьНеобходимостьПерерасчетовДокументовСреднегоЗаработка(ДанныеДляРегистрацииПерерасчетов);
```

Обычно перерасчет регистрируется автоподписками при записи наборов записей регистров среднего заработка. При записи движений в обход бизнес-логики (`ЗарплатаКадры.ОтключитьБизнесЛогикуПриЗаписи`, временный набор, документ ПереносДанных) автоподписка НЕ срабатывает - регистрировать перерасчет нужно явно.

## 38. Начисления: план видов расчета и регистр расчета

- `ПланВидовРасчета.Начисления` - виды расчета начислений.
- `РегистрРасчета.Начисления` - результаты расчета.
- `РегистрСведений.ЗначенияПоказателейНачислений` - значения показателей начислений.

Исторические записи отпусков по периодам и видам доступны через `УчетНачисленнойЗарплатыРасширенный.НачисленныеОтпускаПоПериодамИВидам` (данные из регистра расчета Начисления + показатели из РС).

Нюансы:
- Поля `Показатель1..N` / `Значение1..N` - внутренние поля менеджера расчета, в самом регистре их нет.
- Перечисление `КатегорииНачисленийИНеоплаченногоВремени` для отпуска использует значение `ОплатаОтпуска` (не "Отпуск").

## 39. Корректировка движений через ПереносДанных

Технический непроводимый документ `Документ.ПереносДанных` пишет произвольные движения в любые регистры ЗУП в обход бизнес-логики:

```bsl
Для Каждого ИмяРегистра Из СписокРегистров Цикл
    ДокПеренос.Движения[ИмяРегистра].Загрузить(ТаблицаДвижений[ИмяРегистра]);
    ДокПеренос.ТаблицаРегистров.Добавить().Имя = ИмяРегистра;
КонецЦикла;
ДокПеренос.Записать(); // в транзакции
```

Пишет в обход бизнес-логики - зависимые автоподписки (например перерасчет среднего заработка, см. §37) нужно инициировать явно. Типовой безопасный способ править регистры без перепроведения первичных документов.

## 40. Регламентированная и статистическая отчетность

Документ `Документ.РегламентированныйОтчет`:
- реквизит `ДанныеОтчета` - тип `ХранилищеЗначения`;
- внутри: `ПоказателиОтчета.ПолеТабличногоДокументаФормаОтчета` - плоские ячейки; `ДанныеМногострочныхРазделов` - Структура {ПрефиксРаздела: ТаблицаЗначений};
- реквизита `Номер` у документа нет - искать по Ссылке/Дате.

Конвертация показателей статистической отчетности: `РегламентированнаяОтчетность.КонвертированныеЗначенияПоказателейСтатистическойОтчетности` (деление на делитель данных ИБ, округление). Данные форм статистики персонала - модуль `СтатистикаПерсоналаРасширенный`.

## 41. Пособия и обмен с СФР (СЭДО)

Документы больничных/пособий и электронного обмена с СФР:
- `Документ.БольничныйЛист`
- `Документ.ОтветНаЗапросФССДляРасчетаПособия`, `Документ.ВходящийЗапросФССДляРасчетаПособия`
- `Документ.ЗапросСтатусовВыплатыПособия` (сообщение 8410), `Документ.СтатусыВыплатыПособия` (8510), `Документ.УведомлениеОСтатусеВыплатыПособия` (11000)
- расчетный модуль `УчетПособийСоциальногоСтрахования`

Режим "Сдвигать оплату больничного при наличии исключаемых периодов" передается ключом `СдвигатьПериодОплаты` в структуре `Реквизиты` при формировании ответа в СФР.

## 42. Регистры расчетов с сотрудниками и взносов

- `РегистрНакопления.ЗарплатаКВыплате` - ресурс называется `СуммаКВыплате` (не "Сумма"), ВидДвижения Приход/Расход, реквизиты `ДокументОснование` и `Регистратор`. Закрытие обязательства надежнее определять по типу регистратора, а не по знаку/виду движения.
- Взносы: `РегистрНакопления.СтраховыеВзносыПоФизическимЛицам`, `РегистрНакопления.СведенияОДоходахСтраховыеВзносы`, `Документ.ПерерасчетСтраховыхВзносов`.
- `ОтражениеЗарплатыВБухучете.СформироватьДвиженияПоДокументу` перезатирает движения - дозаполнять после ее вызова.

## Ключевые нюансы

1. **ТолькоРазрешенные=Ложь** в серверном коде документов (привилегированный доступ). `Истина` - для отображения пользователю (RLS).
2. **ДатаПолученияДанных='00010101'** - последние данные. Конкретная дата - актуальные на нее.
3. **Для массовых операций** предпочтительнее массовые методы (`ОсновныеСотрудникиФизическихЛиц`) вместо циклов по единичным (`ОсновнойСотрудникФизическогоЛица`).
4. **Кеширование** `КадровыйУчетПовтИсп` - на время серверного вызова.
5. **Расчетная часть** (средний заработок, начисления, пособия/СФР, взносы, регламентированная отчетность) - см. §36-42. Прочие механизмы, не покрытые справочником, - искать через `templatesearch`.
