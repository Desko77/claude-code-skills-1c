# 1c-project-init — Инициализация проекта 1С

## Когда использовать

- Настройка нового 1С-проекта для работы с Claude Code
- Добавление скилов/правил к существующему проекту
- Обновление устаревших скилов в проекте

Триггеры: "инициализируем проект", "init project", "настрой проект для Claude"

---

## Определение режима

- `.claude/skills/` существует в целевом проекте — режим **enrich** (обновить недостающие/устаревшие)
- Нет `.claude/skills/` — режим **new** (полная инициализация)

---

## Шаги

### 1. Определить целевой путь

- Если указан аргумент — использовать его
- Иначе — текущая рабочая директория

### 2. Сбор информации о проекте

Запросить у пользователя:

1. **Тип проекта**: конфигурация / расширение / обработка / другое
2. **Имя расширения** (если расширение)
3. **Имя базы 1С** (например "Бухгалтерия", "УНФ")
4. **Какие MCP-серверы доступны**: 1c-edt / bsl-platform-context / нет

### 3. Копирование скилов и правил

Из набора `claude-code-skills-1c` скопировать в `<целевой проект>/.claude/`:

#### Обязательные (всегда)
- `rules/1c-coding-standards.md`
- `rules/anti_patterns.md`
- `rules/query-optimization-tips.md`
- `rules/code-review-checklist.md`

#### По типу проекта
- Конфигурация: `skills/1c-meta-compile/`, `skills/1c-cf-*`, `skills/1c-form-*`
- Расширение: `skills/1c-cfe-*`, `rules/1c-extension-patterns.md`
- Обработка: `skills/1c-epf-*`

#### При наличии MCP
- 1c-edt: `skills/1c-edt-tools/`, `rules/bsl-validation-workflow.md`
- bsl-platform-context: добавить ссылки в CLAUDE.md

### 4. Генерация CLAUDE.md

Создать `CLAUDE.md` в корне целевого проекта:

```markdown
# <Имя проекта>

## Тип
<конфигурация / расширение / обработка>

## Зона разработки
<src/ для конфигурации, ext/<ИмяРасширения>/ для расширения>

## Стандарты
- Правила кода: `.claude/rules/1c-coding-standards.md`
- Антипаттерны: `.claude/rules/anti_patterns.md`
- Оптимизация запросов: `.claude/rules/query-optimization-tips.md`

## MCP-серверы
<список доступных серверов>
```

### 5. Режим enrich

- Сравнить файлы по содержимому — копировать только если версия в наборе новее или файл отсутствует
- Никогда не перезаписывать: `CLAUDE.md`, `.mcp.json`, `.v8-project.json`

---

## Важно

Предпочитать скилы ручным командам 1С:

| Задача | Неправильно | Правильно |
|--------|------------|-----------|
| Выгрузка XML | `1cv8 DESIGNER /DumpConfigToFiles` | Скил `1c-db-dump-xml` |
| Загрузка XML | `1cv8 DESIGNER /LoadConfigFromFiles` | Скил `1c-db-load-xml` |
| Создание объекта | Ручное создание XML | Скил `1c-meta-compile` |
| Сборка CF | Ручной 1cv8 | Скил `1c-db-dump-cf` |

---

## Отчёт

После завершения вывести:
- Режим: new / enrich
- Скопированные скилы и правила (список)
- Созданные файлы
- Рекомендации по настройке MCP (если нет)
