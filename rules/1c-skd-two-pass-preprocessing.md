# Двухпроходный СКД: предобработка деталей до свёртки

Применяется: при доработке СКД-отчётов, где нужно отфильтровать/трансформировать ДЕТАЛЬНЫЕ записи ДО того, как СКД свернёт их в группировки. Типовая задача - скрыть строки/столбцы кросс-таба с нулевым свёрнутым итогом.

Кейс - скрыть строки/столбцы с нулевым свёрнутым итогом (Янв -1200 + Март +1200 = 0): ячейка пустая, но каркас группировки остаётся. Запросом/ИМЕЮЩИЕ/отбором СКД не решается (свёртка позже группировки). Постобработка таблдока `Видимость=Ложь` проще, но скрывает шапки группировок и оставляет следы.

## Алгоритм (гибрид)

ДВА прохода в `ПриКомпоновкеРезультата` (`СтандартнаяОбработка = Ложь`):

**Проход 1 - получить плоскую детальную ТЗ.** Вариант А (отдельная схема формирования): клон запроса осн. схемы + плоский вариант настроек (детальные записи, все поля). Вариант Б (без отдельной схемы): та же схема, но строить плоские настройки. Вывод через `ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений` + `ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений` -> `ТаблицаЗначений`.

**Обработка ТЗ** (фильтр нулей / трансформация).

**Проход 2 - вывод кросс-таба.** Схема вывода ОБЯЗАТЕЛЬНО остаётся набором-ЗАПРОСОМ (не конвертировать в объект в файле схемы!). `КомпоновщикМакета.Выполнить(СхемаВывода, Настройки, Расшифровка)` компилирует макет (поля типизированы из запроса -> навигация по ссылкам разрешена). Затем РАНТАЙМ-подмена набора на объект В СКОМПИЛИРОВАННОМ макете + `Инициализировать(Макет, Новый Структура("Результат", ТЗ), Расшифровка, Истина)`.

Рантайм-подмена набора:
```
СтарыйНабор = Макет.НаборыДанных[0];
НовыйНабор = Макет.НаборыДанных.Добавить(Тип("НаборДанныхОбъектМакетаКомпоновкиДанных"));
НовыйНабор.ИмяОбъекта = "Результат"; НовыйНабор.Имя = СтарыйНабор.Имя; НовыйНабор.ИсточникДанных = СтарыйНабор.ИсточникДанных;
Для Каждого П Из СтарыйНабор.Поля Цикл ЗаполнитьЗначенияСвойств(НовыйНабор.Поля.Добавить(), П); КонецЦикла;
ЗаполнитьЗначенияСвойств(НовыйНабор, СтарыйНабор);
Макет.НаборыДанных.Удалить(СтарыйНабор);
```

## Особенности использования

1. **Кросс-таб (Строки+Колонки) НЕ выводится в коллекцию** - только плоский (детальные записи). Поэтому проход 1 нужен с ПЛОСКИМИ настройками.
2. **Набор-ОБЪЕКТ в ФАЙЛЕ схемы (DataSetObject) ломает навигацию по ссылкам.** Поля объекта нетипизированы -> `КомпоновщикМакета.Выполнить` падает «Поле не найдено» на КОМПИЛЯЦИИ макета (до данных). Решение - не конвертировать схему вывода в объект, а подменять набор в РАНТАЙМЕ на скомпилированном макете (там типы уже разрешены из запроса).
3. **Параметры-выражения и генератор коллекции.** Производные параметры схемы (`НачалоПериода=&Период.ДатаНачала`, `ТекущаяДата=ТекущаяДатаСеанса()`) - `ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений` эти ВЫРАЖЕНИЯ НЕ парсит -> «Синтаксическая ошибка ТекущаяДатаСеанса» на `Инициализировать`. Решение: задать производные параметры ЯВНЫМИ конкретными значениями в коде (`Параметр.Значение = ТекущаяДатаСеанса()`), период взять из настроек вывода.
4. **НЕ переносить параметры из настроек целиком** (`Перенести все ПараметрыДанных.Элементы`) - вывод хранит производные как `ВыражениеКомпоновкиДанных` и они утекают в проход 1. Брать ТОЛЬКО `Период` (СтандартныйПериод), остальное вычислять.
5. **`ПараметрыДанных.НайтиЗначениеПараметра` ждёт `ПараметрКомпоновкиДанных`, НЕ строку.** Передача строки -> «Несоответствие типов (параметр 1)». Оборачивать: `НайтиЗначениеПараметра(Новый ПараметрКомпоновкиДанных(ИмяСтрокой))`. (А `УстановитьЗначениеПараметра` строку принимает - асимметрия.)
6. **Ресурсные колонки определять по `Схема.ПоляИтога`, НЕ по типу колонки ТЗ.** Суммы из выгрузки запроса имеют СОСТАВНОЙ тип `Null+Число` (левые соединения) -> «ровно Число» их отбрасывает, отчёт пустеет. `Для Каждого ПолеИтога Из Схема.ПоляИтога Цикл ... Строка(ПолеИтога.ПутьКДанным) ...`.
7. **Значения скомпилированных параметров - в `МакетКомпоновки.ЗначенияПараметров`** (не в `НаборДанных.ЗначенияПараметров` - там пусто). У набора-запроса макета свойство `ЗначенияПараметров` (не `Параметры`).
8. **`ПолеНабораДанныхМакетаКомпоновкиДанных`** имеет только `Имя/ПутьКДанным/Роль` (нет `ТипЗначения`). Колонки в ТЗ дополнять по `.Имя`.
9. **EDT НЕ подхватывает внешние правки .dcs** (через Python/Bash/git checkout) - в build-модель не попадают без `clean_project`/refresh. При деплое в дев-контур ФАЙЛЫ схем переносить ОТДЕЛЬНО от BSL-модуля; новый макет регистрировать в `.mdo` (`<templates>` с уникальным UUID).
10. **Семантика фильтра нулей:** удалять детальную строку ТЗ только когда ВСЕ ресурсы (ПоляИтога) нулевые; частично заполненные оставлять. Свёртка оставшихся даёт корректный кросс-таб без пустых строк/колонок.

## Когда применять
Кросс-таб СКД-отчёт, где нужна предобработка детальных данных до свёртки (фильтр нулевых свёрток, дедуп, перерасчёт). Если хватает постобработки таблдока `Видимость=Ложь` - проще она, но она скрывает шапки группировок и оставляет следы.
