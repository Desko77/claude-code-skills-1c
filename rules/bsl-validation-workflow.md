# Workflow валидации BSL-кода

Применяется: при генерации, редактировании или валидации кода 1С (BSL).

Этот файл описывает **порядок валидации** — когда и в какой последовательности проверять код.
- Что проверять — см. `code-review-checklist.md`
- Справочник инструментов EDT — см. `skills/1c-edt-tools/SKILL.md`

---

## Принципы

1. **Не вызывать API без проверки** — перед использованием любого метода/свойства платформы убедиться в его существовании через `get_platform_documentation` (1c-edt) или `getMembers` (bsl-platform-context)
2. **Не использовать метаданные без верификации** — перед обращением к объектам (Справочники.*, Документы.* и т.д.) проверить их наличие через `get_metadata_details` / `get_metadata_objects` (1c-edt) или `search_metadata` (bsl-platform-context)
3. **Искать существующие паттерны** — перед написанием нового кода найти аналогичные реализации через `search_in_code` (1c-edt) или `search_code` (bsl-platform-context)
4. **Линтить после записи** — после каждой записи .bsl-файла запускать проверку и исправлять ошибки

---

## Workflow ДО написания кода

### Шаг 1. Поиск паттернов

Найти существующие реализации аналогичной логики:
- `search_in_code` (1c-edt) — полнотекстовый поиск по BSL-модулям
- `search_code` (bsl-platform-context, опционально) — поиск процедур/функций по описанию

Если найден подходящий паттерн — использовать как основу.

### Шаг 2. Валидация API платформы

Для каждого используемого типа платформы (СправочникМенеджер, ДокументОбъект, ТаблицаЗначений и т.д.):
- `get_platform_documentation` (1c-edt) — документация по типу, методам, свойствам
- `getMembers` (bsl-platform-context, опционально) — список доступных методов/свойств

Если метод не найден — остановиться и сообщить пользователю с перечнем доступных альтернатив.

### Шаг 3. Валидация метаданных

Для каждого объекта метаданных (Справочники.*, Документы.*, Регистры.* и т.д.):
- `get_metadata_details` (1c-edt) — реквизиты, табличные части, формы
- `get_metadata_objects` (1c-edt) — поиск объекта по имени
- `search_metadata` (bsl-platform-context, опционально) — структура объекта

Проверить наличие всех используемых реквизитов и табличных частей.

### Шаг 4. Валидация запросов

Если код содержит запрос на языке запросов 1С:
- Извлечь имена таблиц и полей из текста запроса
- Проверить каждое имя через `get_metadata_details`
- `validate_query` (1c-edt) — синтаксис и семантика запроса
- Для запросов СКД — `validate_query` с `dcsMode=true`

---

## Workflow ПОСЛЕ написания кода

### Шаг 5. Проверка ошибок

Сразу после записи .bsl-файла:
- `get_project_errors` (1c-edt) — ошибки проекта с фильтром по объекту
- `revalidate_objects` (1c-edt) — перевалидация если нужно обновить кеш

### Шаг 6. Цикл исправления

Если обнаружены ошибки уровня ERRORS/BLOCKER/CRITICAL:
1. Проанализировать ошибки (`get_check_description` для деталей)
2. Для ошибок API — найти правильный метод через `get_platform_documentation`
3. Исправить код и перезаписать файл
4. Повторить проверку (максимум 3 итерации)
5. Если ошибки остаются — остановиться и сообщить пользователю

## Таблица инструментов

| Задача | 1c-edt (основной) | bsl-platform-context (опционально) |
|--------|--------------------|------------------------------------|
| Поиск кода | `search_in_code` | `search_code` |
| Документация API | `get_platform_documentation` | `getMembers`, `getConstructors` |
| Структура метаданных | `get_metadata_details` | `search_metadata` |
| Поиск метаданных | `get_metadata_objects` | `search_metadata_by_description` |
| Проверка ошибок | `get_project_errors` | — |
| Валидация запросов | `validate_query` | — |
| Описание ошибки | `get_check_description` | — |
| Перевалидация | `revalidate_objects` | — |

Инструменты `bsl-platform-context` опциональны — при наличии MCP-сервера bsl-platform-context. Все задачи можно решить через `1c-edt`.

---

## Условия остановки

Прекратить генерацию кода и запросить помощь пользователя если:

- Метод/свойство платформы не найден ни в одном источнике
- Объект метаданных не существует в конфигурации
- Реквизит/табличная часть отсутствует в структуре объекта
- Ошибки не устранены после 3 итераций исправления
- MCP-сервер недоступен и валидация невозможна
