# Программная работа с профилями групп доступа БСП

Применяется: BSL, БСП 3.x (ЗУП 3.1, БП 3.x, ERP 2.x, УТ 11.x). Программное создание/обновление справочника `ПрофилиГруппДоступа` через консоль кода или код обработчиков обновления, назначение профилей пользователям, проверки прав, ролей и RLS-доступа.

---

## Тип реквизита `Роли.Роль` - НЕ строка

В БСП 3.x реквизит `Справочник.ПрофилиГруппДоступа.Роли.Роль` имеет тип **`СправочникСсылка.ИдентификаторыОбъектовМетаданных`**, а не строка с именем роли.

Если присвоить строку:
```bsl
// НЕПРАВИЛЬНО - значение молча отбрасывается
НоваяСтрока = Профиль.Роли.Добавить();
НоваяСтрока.Роль = "ПолныеПрава";
```

Симптом: `Профиль.Записать()` проходит без ошибок, но в ТЧ оказывается 0-1 строка вместо ожидаемого N. БСП-обработчик при записи фильтрует невалидные значения.

## Правильный путь - ссылка на ИдентификаторыОбъектовМетаданных

`ПолноеИмя` записи справочника для роли: `"Роль." + ИмяРоли`.

Массовое получение ссылок через запрос (предпочтительно для пакетной обработки):

```bsl
ПолныеИмена = Новый Массив;
Для Каждого ИмяРоли Из ИменаРолей Цикл
    ПолныеИмена.Добавить("Роль." + ИмяРоли);
КонецЦикла;

Запрос = Новый Запрос;
Запрос.Текст =
"ВЫБРАТЬ РАЗРЕШЕННЫЕ
|	Идентификаторы.ПолноеИмя КАК ПолноеИмя,
|	Идентификаторы.Ссылка КАК Ссылка
|ИЗ
|	Справочник.ИдентификаторыОбъектовМетаданных КАК Идентификаторы
|ГДЕ
|	Идентификаторы.ПолноеИмя В (&ПолныеИмена)
|	И НЕ Идентификаторы.ПометкаУдаления";
Запрос.УстановитьПараметр("ПолныеИмена", ПолныеИмена);

КартаИдентификаторов = Новый Соответствие;
Выборка = Запрос.Выполнить().Выбрать();
Пока Выборка.Следующий() Цикл
    КартаИдентификаторов.Вставить(Выборка.ПолноеИмя, Выборка.Ссылка);
КонецЦикла;
```

Точечное получение ссылки (БСП API, создает запись справочника при отсутствии):
```bsl
СсылкаИдент = ОбщегоНазначения.ИдентификаторОбъектаМетаданных(Метаданные.Роли["ИмяРоли"]);
```

## Роли и объекты РАСШИРЕНИЯ - отдельный справочник ИдентификаторыОбъектовРасширений

Объекты расширений (роли, справочники, документы) регистрируются НЕ в `ИдентификаторыОбъектовМетаданных`,
а в отдельном справочнике `ИдентификаторыОбъектовРасширений`. Реквизит `ПрофилиГруппДоступа.Роли.Роль` -
составного типа (`CatalogRef.ИдентификаторыОбъектовРасширений` + `CatalogRef.ИдентификаторыОбъектовМетаданных`).

**Прямой запрос к `Справочник.ИдентификаторыОбъектовМетаданных` (пример выше) роли расширения НЕ находит** -
профиль получит только базовые роли, роли расширения молча пропущены. Для конфигураций С РАСШИРЕНИЯМИ
использовать метод БСП, обрабатывающий ОБА справочника:

```bsl
ПолныеИмена = Новый Массив;
Для Каждого ИмяРоли Из ИменаРолей Цикл
    ПолныеИмена.Добавить("Роль." + ИмяРоли);
КонецЦикла;
// Возвращает Соответствие [ПолноеИмя -> ссылка] (метаданных ИЛИ расширений).
// ВызыватьИсключение=Ложь -> ненайденные пропускаются.
КартаИдентификаторов = ОбщегоНазначения.ИдентификаторыОбъектовМетаданных(ПолныеИмена, Ложь);
СсылкаИдент = КартаИдентификаторов.Получить("Роль." + ИмяРоли);
```

В списке ролей профиля (UI) роли отображаются по СИНОНИМУ, не по имени - фильтр по имени ("Расш_") не найдет
роль с человекочитаемым синонимом. Искать по синониму или заполнять программно.

## Если ссылок нет в справочнике

Если для какой-то роли ссылка в `ИдентификаторыОбъектовМетаданных` отсутствует (новая роль добавлена в конфигурацию, но БСП не успел зарегистрировать) - запустить обработчик обновления типовой подсистемы БСП "Идентификаторы объектов метаданных" (модуль `ОбновлениеВерсииИБ` / `ОбщегоНазначенияБТСПереопределяемый`) или вызвать `ОбщегоНазначения.ИдентификаторОбъектаМетаданных()` - он создаст запись.

## Шаблон создания/обновления профиля

```bsl
УстановитьПривилегированныйРежим(Истина);

ИмяПрофиля = "Имя профиля";

// 1. Собрать имена ролей в ИменаРолей (Массив)
// 2. Получить ссылки на идентификаторы (см. блок выше) -> КартаИдентификаторов

// 3. Найти/создать профиль
СсылкаПрофиля = Справочники.ПрофилиГруппДоступа.НайтиПоНаименованию(ИмяПрофиля, Истина);
Если СсылкаПрофиля.Пустая() Тогда
    Профиль = Справочники.ПрофилиГруппДоступа.СоздатьЭлемент();
    Профиль.Наименование = ИмяПрофиля;
Иначе
    Профиль = СсылкаПрофиля.ПолучитьОбъект();
    Если Профиль.ПоставляемыйПрофиль Тогда
        ВызватьИсключение "Поставляемый профиль - редактирование запрещено";
    КонецЕсли;
КонецЕсли;

// 4. Заполнить ТЧ Роли ссылками (НЕ строками)
Профиль.Роли.Очистить();
Для Каждого ИмяРоли Из ИменаРолей Цикл
    СсылкаИдент = КартаИдентификаторов.Получить("Роль." + ИмяРоли);
    Если СсылкаИдент = Неопределено Тогда
        Продолжить;
    КонецЕсли;
    НоваяСтрока = Профиль.Роли.Добавить();
    НоваяСтрока.Роль = СсылкаИдент;
КонецЦикла;

// 5. Назначение (Пользователи / ВнешниеПользователи) - заполнить, если пусто
Если Профиль.Назначение.Количество() = 0 Тогда
    Назн = Профиль.Назначение.Добавить();
    Назн.ТипПользователей = Справочники.Пользователи.ПустаяСсылка();
КонецЕсли;

// 6. Запись - БЕЗ ОбменДанными.Загрузка = Истина
// (иначе БСП-обработчики не отработают - регистрация в подсистеме УправлениеДоступом, обновление кэша прав)
Профиль.Записать();
```

## Назначение профиля пользователю программно

Стабильный API модуля `УправлениеДоступом` (область `ПрограммныйИнтерфейс`, сервер):

```bsl
// Включить профиль: ссылка ПрофилиГруппДоступа, УникальныйИдентификатор
// поставляемого профиля или его строковое имя. В упрощенном режиме настройки прав
// создает/находит персональную группу доступа и включает в нее пользователя.
УправлениеДоступом.ВключитьПрофильПользователю(Пользователь, Профиль);

// Выключить профиль; Профиль = Неопределено - отключить все профили пользователя
УправлениеДоступом.ВыключитьПрофильПользователю(Пользователь, Профиль = Неопределено);

// Полная переустановка прав: массивы групп доступа (или профилей) и групп пользователей
УправлениеДоступом.УстановитьПраваПользователя(Пользователь, ГруппыДоступа, ГруппыПользователей);
```

`Пользователь` - `СправочникСсылка.Пользователи` или `СправочникСсылка.ВнешниеПользователи`. Метода вида `УправлениеДоступом.ДобавлениеПользователейВГруппу` в БСП нет: назначение профиля - `ВключитьПрофильПользователю`, полная переустановка - `УстановитьПраваПользователя`. `ВключитьПрофильПользователю` рассчитан на упрощенный режим настройки прав; в неупрощенном режиме работать с группами доступа через `УстановитьПраваПользователя`.

## Сопоставление "имена ролей по синониму"

Если входные данные - синонимы ролей (русские названия из xlsx, документации), а не имена для разработчика:

```bsl
КартаПоСинониму = Новый Соответствие;
КартаПоСинонимуНорм = Новый Соответствие;
Для Каждого МетаРоль Из Метаданные.Роли Цикл
    КартаПоСинониму.Вставить(МетаРоль.Синоним, МетаРоль.Имя);

    Норм = НРег(СокрЛП(МетаРоль.Синоним));
    Пока СтрНайти(Норм, "  ") > 0 Цикл
        Норм = СтрЗаменить(Норм, "  ", " ");
    КонецЦикла;
    КартаПоСинонимуНорм.Вставить(Норм, МетаРоль.Имя);
КонецЦикла;
```

Двухуровневый матчинг: точное совпадение по `Синоним`, при промахе - по нормализованному (нижний регистр + СокрЛП + сжатие двойных пробелов). Покрывает разнобой в исходных данных.

## Проверка права, роли и RLS-доступа

Точные сигнатуры (все - сервер, область `ПрограммныйИнтерфейс`):

```bsl
// Роли конфигурации: имена через запятую СТРОКОЙ (не массив).
// Истина - если доступна хотя бы одна; полноправному - Истина
// при УчитыватьПривилегированныйРежим = Истина
Пользователи.РолиДоступны(ИменаРолей, Пользователь = Неопределено, УчитыватьПривилегированныйРежим = Истина)

// Полные права (опционально вместе с ролью АдминистраторСистемы)
Пользователи.ЭтоПолноправныйПользователь(Пользователь = Неопределено, ПроверятьПраваАдминистрированияСистемы = Ложь, УчитыватьПривилегированныйРежим = Истина)

// Право на конкретный объект-владелец прав (папку файлов и т.п.), с учетом иерархии.
// СсылкаНаОбъект - ссылка на объект данных, НЕ объект метаданных
УправлениеДоступом.ЕстьПраво(Право, СсылкаНаОбъект, Знач Пользователь = Неопределено)

// Роль в профилях групп доступа (с учетом RLS на чтение)
УправлениеДоступом.ЕстьРоль(Знач Роль, Знач СсылкаНаОбъект = Неопределено, Знач Пользователь = Неопределено)

// RLS: разрешено ли чтение/изменение на уровне записей - Функции -> Булево
УправлениеДоступом.ЧтениеРазрешено(ОписаниеДанных, Пользователь = Неопределено)
УправлениеДоступом.ИзменениеРазрешено(ОписаниеДанных, Пользователь = Неопределено)
// Те же проверки, но с исключением при запрете - Процедуры
УправлениеДоступом.ПроверитьЧтениеРазрешено(ОписаниеДанных)
УправлениеДоступом.ПроверитьИзменениеРазрешено(ОписаниеДанных)
```

Особенности:

- Платформенный `РольДоступна("ПолныеПрава")` не учитывает привилегированный режим и полноправность - в конфигурациях на БСП использовать `Пользователи.РолиДоступны` / `ЭтоПолноправныйПользователь`.
- `ЕстьРоль` проверяет роль в профилях групп доступа с учетом RLS на чтение; проверка роли конфигурации без RLS - `Пользователи.РолиДоступны`.
- В `ЕстьПраво` второй аргумент - ссылка (например, папка файлов); передача `Метаданные.*` - ошибка. Метода `УправлениеДоступом.НастройкиПрав` не существует: состав возможных прав задается хуком `УправлениеДоступомПереопределяемый`.
- `ЧтениеРазрешено` / `ИзменениеРазрешено` - про ограничения на уровне записей (RLS). В стандартном (не производительном) варианте RLS указание пользователя, отличного от текущего, вызывает исключение - предварительно проверять `УправлениеДоступом.ПроизводительныйВариант()`.
- Клиентский аналог полных прав - `ПользователиКлиент.ЭтоПолноправныйПользователь(ПроверятьПраваАдминистрированияСистемы = Ложь)`, только для текущего пользователя.

## Анти-паттерны

- **Присвоение строки в `Роль`** - молча игнорируется БСП-обработчиком при записи.
- **`Профиль.ОбменДанными.Загрузка = Истина` перед `Записать()`** - отключает БСП-обработчики (регистрация в УправлениеДоступом, обновление прав), профиль записывается, но в системе прав не регистрируется.
- **Редактирование `ПоставляемыйПрофиль = Истина`** - запись возможна, но при следующем обновлении конфигурации БСП восстановит исходное состояние из поставки. Нужно создать пользовательский профиль с другим именем.
- **`УстановитьПривилегированныйРежим` без выключения** - стандартное правило, но в разовой консоли кода не критично.
